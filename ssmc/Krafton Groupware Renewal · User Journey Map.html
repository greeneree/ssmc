P1
실무자
사원·대리급 / 내부 인증 / 전체 기능 접근
그룹웨어 핵심 사용자, 반복 업무 빈도 최다
"결재가 어디까지 갔는지 알림도 없이 직접 들어가 확인해야 해."
Scenario
휴가 신청 → Workflow 기안 → 결재 상태 확인
일상적인 반복 업무(휴가 신청 + 기안 작성)를 수행하고 결재 진행 상태를 확인하는 최빈도 시나리오. 현재 Workflow가 외부 SaaS로 분리된 구조에서 발생하는 시스템 전환 비용이 핵심 Pain.
Stage 1

Workflow 기안		 Stage 4
결재 상태 확인		 Stage 5
완료 및 이탈
터치포인트 그룹웨어 홈 화면
Web/Mobile		 휴가 신청 메뉴
그룹웨어 내		 외부 Workflow SaaS
별도 시스템 전환		 그룹웨어 복귀 후 알림 확인 또는 Workflow 재진입 그룹웨어 홈 복귀
알림 수신(이메일/푸시)
사용자 행동 로그인 → 홈 대시보드 확인 → 오늘 할 일·알림 체크 잔여 휴가 일수 확인 → 별도 신청 화면 이동 → 날짜 입력 → 제출 그룹웨어 이탈 → 외부 Workflow 시스템 접속 → 양식 검색 → 기안 작성 → 제출 그룹웨어 복귀 → 알림 없음 확인 → Workflow 시스템 재접속 → 결재 현황 탭 클릭 결재 완료 알림 수신 → 그룹웨어에서 최종 상태 재확인
페인 포인트
  • 홈 화면에 오늘 내 결재·휴가 현황이 없어 직접 탐색 필요
  • 기능별 UI 패턴이 달라 매번 새로 적응
  • 잔여 일수 확인 화면 → 신청 화면 2단계 이동 강제
  • 팀원 휴가 일정 충돌 여부를 신청 전에 알 수 없음
  • 그룹웨어 ↔ 외부 시스템 전환으로 업무 맥락 단절
  • Workflow 시스템에서 적합한 양식을 찾는 데 시간 소요
  • 기안 중 그룹웨어 데이터(부서·직급 등) 자동 연동 안 됨
  • 결재 진행 상태를 그룹웨어에서 확인 불가
  • 상태 확인 위해 외부 시스템 재접속 반복
  • "내가 제대로 제출한 게 맞나" 불안 지속
  • 최종 완료 알림이 어느 채널로 오는지 불명확
  • 완료 후 그룹웨어에 결과 반영 여부 불확실
개선 기회 홈 대시보드 개인화: 미처리 결재·잔여 휴가·팀 일정을 한눈에 인라인 컨텍스트: 신청 폼 내 잔여 일수 + 팀원 일정 충돌 여부 동시 표시 Workflow 그룹웨어 통합: 기안·결재를 그룹웨어 이탈 없이 처리. 인사 데이터 자동 연동 결재 타임라인 실시간 표시: 현재 단계·담당자·예상 처리기간을 홈 또는 신청 완료 화면에서 노출 처리 완료 푸시 알림 + 그룹웨어 홈 자동 갱신
UX 설계 방향
1
일관된 인터랙션 패턴 통일
신청·결재·조회 기능 전반에 동일한 폼 구조, 버튼 위치, 상태 표시 방식 적용 → 학습 비용 제거
2
결재 상태 실시간 노출
홈 또는 신청 완료 화면에서 현재 결재 단계·담당자·예상 처리기간을 타임라인으로 표시
3
인라인 컨텍스트 제공
휴가 신청 폼 내 잔여일수, 팀 일정 충돌 여부를 한 화면에서 동시 제공
4
Workflow 통합 모듈
외부 SaaS 전환 없이 그룹웨어 내에서 기안·결재 처리. 인사 데이터 자동 연동으로 입력 최소화
P2
관리자
팀장급 / 내부 인증 / 팀 관리 + 결재 권한
결재자이자 팀 현황 책임자
"결재 대기가 10건인데 뭐가 급한지 모르겠어. 위임은 매번 어디 있는지 찾아야 해."
Scenario
팀 현황 파악 → 결재 처리 → 팀원 휴가 관리
관리자는 실무자와 동일한 그룹웨어를 쓰지만 '팀 단위 현황 파악'과 '우선순위 판단'이 목적. 알림(그룹웨어) ↔ 처리(외부 Workflow)의 분리가 핵심 Pain.
Stage 1
접속 및 현황 파악		 Stage 2
결재 대기 확인		 Stage 3
결재 처리		 Stage 4
팀 근태·휴가 관리		 Stage 5
위임·예외 처리
터치포인트 그룹웨어 홈 (개인 뷰) 그룹웨어 알림
→ 외부 Workflow 전환		 외부 Workflow 결재함 그룹웨어 팀원 관리 메뉴 그룹웨어 설정 메뉴
(위임 기능 탐색)
사용자 행동 로그인 → 홈 확인 → 개인 업무 중심 화면에서 팀 현황 직접 탐색 그룹웨어 알림 수신 → 외부 Workflow 시스템 접속 → 결재함 확인 10건의 결재 목록 확인 → 우선순위 판단 시도 → 순서대로 처리 팀원별 휴가 신청 내역 확인 → 일정 충돌 수동 확인 → 승인/반려 부재 시 위임 설정 필요 → 메뉴 위치 탐색 → 3depth 이후 접근
페인 포인트
  • 홈이 개인 뷰로 고정 → 팀 현황을 보려면 추가 탐색 필요
  • 실무자와 동일한 화면 구조로 관리자 맥락 정보 부재
  • 알림은 그룹웨어, 처리는 외부 시스템 — 채널 분리로 인지 부담 증가
  • 시스템 전환 과정에서 맥락 손실
  • 결재 목록이 접수 순서로만 정렬 → 우선순위 판단 불가
  • 기한·중요도·연관 업무 정보 목록 내 미노출
  • 팀원 휴가 일정 충돌 여부를 달력 형태로 볼 수 없음
  • 이상 근태(지각·조퇴 반복 등) 사후 파악
  • 결재 위임 기능이 깊은 depth에 위치 → 급할 때 못 찾음
  • 결재 공백 발생 위험
개선 기회 관리자 전용 홈 뷰: 팀 근태·미처리 결재·휴가 현황 상단 우선 배치 Workflow 통합 후: 그룹웨어 홈에서 알림 수신 → 처리까지 원스톱 완결 결재함 우선순위 시각화: 기한 임박·긴급 태그 인라인 표시 + 정렬 옵션 팀원 휴가 캘린더 뷰 + 이상 근태 자동 감지 알림 결재 위임·권한 변경을 홈 또는 프로필에서 1depth 접근
UX 설계 방향
1
팀 중심 홈 뷰 분리
관리자 홈에는 개인 업무보다 팀 근태·휴가·미처리 결재 현황을 상단에 우선 배치
2
결재함 우선순위 시각화
기한 임박·긴급 태그를 결재 목록에 인라인 표시하고 정렬 옵션 제공
3
관리 기능 빠른 접근
결재 위임·팀원 권한 변경 등을 홈 또는 프로필 영역에서 1depth로 접근 가능하게 배치
4
Workflow 통합 원스톱 처리
알림 수신부터 결재 처리까지 시스템 전환 없이 그룹웨어 내에서 완결
P3
인사담당자
HR·총무·복지 Admin / 시스템 전체 관리 권한
단일 Admin UI를 다목적으로 사용하는 복수 역할
"입퇴사 처리마다 권한을 수동으로 하나씩 바꾸는 게 제일 힘들어."
Scenario
입사 처리 → 권한 설정 → Workflow Admin 관리
HR·총무·복지 등 다양한 Admin 역할이 동일 시스템을 사용. 예비입사자 계정 사전 생성 불가, Workflow는 별도 외부 SaaS에서 관리하는 이중 운영 구조가 핵심 Pain.
Stage 1
Admin 접속		 Stage 2
신규 입사자 처리		 Stage 3
권한·계정 관리		 Stage 4
Workflow Admin 관리		 Stage 5
전사 현황 모니터링
터치포인트 그룹웨어 Admin 화면 Admin → 계정 생성·권한 설정 부서이동·퇴사 처리 메뉴 외부 Workflow SaaS Admin
별도 시스템		 Admin 통계/보고서 메뉴
사용자 행동 Admin 접속 → HR·총무·복지 설정이 혼재된 화면에서 필요 메뉴 탐색 입사일에 수동으로 계정 생성 → 부서·직급 입력 → 권한 그룹 수동 선택 → 저장 부서이동·퇴사 발생 시 계정별 권한 항목을 하나씩 수동 변경 그룹웨어 이탈 → Workflow SaaS Admin 접속 → Builder·결재선 설정 관리 전사 근태·휴가·결재 현황 조회 → 항목별 개별 메뉴 진입
페인 포인트
  • HR·총무·복지 등 다른 역할의 Admin 설정이 한 화면에 혼재
  • 내 업무 영역 외 설정 항목이 탐색 방해
  • 예비입사자는 현재 접근 불가 → 입사일 전 사전 안내 채널 없음
  • 입사일 수동 처리로 권한 누락·오적용 위험
  • 전환 체크리스트 없어 항목 누락 빈번
  • 부서이동·퇴사 시 권한 변경 항목 多 → 수동 반복 작업
  • 설정 오류 시 보안 이슈 직결
  • 그룹웨어 계정 관리 + Workflow 설정을 별도 시스템에서 이중 운영
  • Workflow 설정 변경 시 영향 범위 파악 어려움
  • 전사 근태·휴가 데이터를 한눈에 볼 수 있는 대시보드 부재
  • 이상 징후 사후 파악 → 선제 대응 불가
개선 기회 역할 기반 Admin 대시보드: HR/총무/복지 각자 자신의 영역만 우선 표시 온보딩 전환 자동화: 예비입사자 계정 사전 예약 생성 + 입사일 자동 권한 전환 + 체크리스트 권한 그룹 자동 추천: 부서·직급 입력 시 권한 그룹 자동 매핑 → 수동 선택 최소화 통합 Admin: 그룹웨어 내에서 Workflow Builder + 계정 + 게시판 설정을 단일 화면에서 관리 전사 현황 Admin 대시보드: 근태 이상·대량 휴가·결재 병목 지표 실시간 모니터링
UX 설계 방향
1
계정 설정 자동화 플로우
부서·직급 입력 시 권한 그룹 자동 추천, 입사일 기반 전환 스케줄 자동 구성
2
역할 기반 Admin 분리
HR·총무·복지 등 Admin 역할별로 접속 시 자신의 관리 영역만 우선 노출
3
전사 현황 대시보드
근태 이상·대량 휴가·결재 병목 등 주요 지표를 한 화면에서 모니터링
4
온보딩 전환 체크리스트
예비입사자→임직원 전환 시 필수 처리 항목을 단계별로 제공하고 누락 자동 감지
P4
외주파트너
외주업체 직원 / 별도 인증 / 협업 범위 제한 접근
현재: 원칙적 접근 제한, 예외 처리 수준
"초대받아서 들어왔는데 어디까지 볼 수 있는 건지 아무도 안 알려줘."
Scenario
접근 권한 요청 → 초대 수락 → 협업 Workspace 진입
현재 공식 진입 경로 자체가 없어 접근 권한 획득 과정(Stage 0)이 가장 큰 Pain. 이번 리뉴얼로 신규 설계되는 경험 영역.

재접속 방법 불명확
사용자 행동 담당자에게 개별 연락 → 권한 요청 → 언제 접근 가능한지 대기 → 담당자 재문의 반복 임시 계정 정보 수신 → 접속 → 임직원과 동일한 전체 메뉴 화면 진입 → 어디로 가야 할지 모름 접근 가능한 메뉴와 불가한 메뉴 구분 없이 탐색 → 오류 화면 반복 진입 공유 문서 열람·업로드 시도 → 내부 용어·프로세스 파악 없이 진행 → 담당자에게 반복 문의 작업 완료 후 로그아웃 → 재접속 시 URL·계정 정보 재확인 필요
페인 포인트
  • 접근 가능 여부 자체가 불명확
  • 계정 발급 공식 프로세스 없음
  • 무엇을 볼 수 있는지 사전 안내 전무
  • 임직원과 동일한 UI로 진입 → 협업에 불필요한 메뉴가 함께 노출
  • 외주파트너 전용 온보딩 가이드 없음
  • 접근 불가 메뉴가 시각적으로 구분되지 않음 → 반복 오류 경험
  • 어떤 행동이 허용되는지 확신 불가 → 행동 위축
  • 크래프톤 내부 용어·약어 사전 지식 없이 시스템 사용
  • 한국어 전용 UI → 외국 국적 외주파트너 즉각적 언어 장벽
  • 다국어 지원 있어도 번역 누락 항목 혼재 (현재 소스코드 하드코딩 구조)
  • 재접속 경로 안내 없음
  • 협업 히스토리 확인 경로 불명확
