화면 구조 개요

어드민 메뉴 실시간 강의 관리 진입 시 탭 2개로 구성된 단일 화면으로 진입한다.
LNB 3depth 분리 아님 — 같은 도메인(Zoom 계정)의 두 뷰를 탭으로 통합.
TAB STRUCTURE 기본 탭: 배정 관리
탭 레이블 기본 진입 목적
TAB_1 배정 관리 ✓ 기본 활성 Zoom 계정 ↔ 강좌 매핑 및 해제. 이번 주 강의 현황 확인.
TAB_2 계정 관리 — Zoom 계정 등록 / 비활성화 / 삭제 / 사용 이력 조회.
⚠ 계정 추가(탭 2) 시 탭 1 헤더 카운트 계정 {N}개 와 탭 2 헤더 카운트가 동시 동기화 필요. 비활성화된 계정은 탭 1 배정 목록에서 노출 제외.

03

상태 정의 및 전이

계정 상태 전이도
STATE TRANSITION
01
등록 (탭 2 계정 추가)
계정 추가 모달 → 등록 버튼 → 활성 상태로 생성. 탭 1 미배정 목록에 자동 추가. 전체/활성 카운트 +1.
02
배정 (탭 1 과정 배정)
미배정 → 과정 배정 드롭다운 → 강좌 선택 → 배정 상태 전환.
03
강의 시작
배정 → 강의 시작 시 진행 중 자동 전환. 이 상태에서 배정 해제 불가.
04
강의 종료 / 배정 해제
강의 종료 → 배정 복귀 (과정 기간 중). 과정 종료 또는 수동 해제 → 미배정 전환.
05
비활성화 (탭 2)
활성 → 비활성화 버튼 → 비활성. 탭 1 배정 목록에서 제거. 기존 배정 있으면 먼저 해제 필요.
06
삭제 (탭 2, 비활성 상태에서만 가능)
비활성 → 삭제 버튼 → 삭제 확인 모달 → 영구 삭제. 모든 이력 함께 삭제. 전체 카운트 -1.
04

모달 명세

계정 추가 모달
MODAL — 계정 추가
  • 트리거 탭 2 우측 상단 "+ 계정 추가" 버튼
  • 계정 ID 텍스트 입력 (예: Zoom06)
  • 이메일 주소 텍스트 입력 (예: aistudio.service+zoom06@multicampus.com)
  • 계정 시작일 Date Picker
  • 만료일 Date Picker
  • 안내 문구 "등록 후 Zoom 마스터 계정에서 해당 유저가 Licensed 상태인지 확인하세요."
  • 버튼 — 등록 등록 처리 → 목록 추가 → 전체/활성 카운트 +1 → 탭 1 미배정 목록 추가
  • 버튼 — 취소 모달 닫힘, 변경 없음
배정 해제 확인 모달
MODAL — 배정 해제 확인
  • 트리거 탭 1 "배정 해제" 버튼 클릭 (상태: 배정)
  • 메시지 "해당 과정의 배정을 해제하시겠습니까?" + 강좌명 표시
  • 버튼 — 취소 모달 닫힘
  • 버튼 — 확인 배정 해제, 상태 미배정으로 전환
비활성화 경고 모달
MODAL — 비활성화 경고
  • 트리거 탭 2 "비활성화" 버튼 클릭 (상태: 활성)
  • 메시지 "비활성화 시 배정 관리에서 선택이 불가합니다. 현재 배정된 과정이 있다면 먼저 해제하세요."
  • 버튼 — 취소 모달 닫힘
  • 버튼 — 비활성화 상태 비활성으로 변경, 탭 1 목록에서 제거
계정 삭제 확인 모달
MODAL — 삭제 확인 (Destructive)
  • 트리거 탭 2 "삭제" 버튼 클릭 (상태: 비활성에서만 노출)
  • 계정명 표시 삭제 대상 계정명 강조 표시
  • 메시지 "삭제 시 해당 계정의 모든 이력이 함께 삭제됩니다. 이 작업은 되돌릴 수 없습니다."
  • 버튼 — 취소 모달 닫힘
  • 버튼 — 삭제 Red/Destructive 스타일. 계정 영구 삭제. 전체 카운트 -1.
중요
삭제는 비활성화 상태에서만 가능. 활성 상태에서는 삭제 버튼 미노출. 삭제 시 사용 이력 포함 모든 데이터 영구 삭제.
사용 이력 모달
MODAL — 사용 이력 조회
  • 트리거 탭 2 목록 "이력 보기" 버튼 클릭
  • 제목 "{계정명} 사용 이력"
  • 컬럼 과정명 / 강의명 / 일시 / 상태 (진행 중 / 완료)
  • 정렬 최신 순 (DESC)
  • 버튼 — 닫기 모달 닫힘
