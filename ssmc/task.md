## 1단계. 파일 읽기
krafton_journey_map.html 파일을 읽고 아래 항목을 추출해:
- 5개 페르소나 (P1~P5)의 이름, 역할, 핵심 인용문
- 각 페르소나별 Stage 목록과 핵심 Pain Point (각 Stage당 1~2개로 압축)
- 각 페르소나별 개선 기회 핵심 메시지
- UX 설계 방향 (페르소나별 핵심 2개)

## 2단계. PPTX 생성
pptxgenjs를 설치하고, 아래 스펙으로 krafton_journey_proposal.pptx를 생성해.

### 전체 디자인 기준
- 슬라이드 크기: 16:9 (13.33 x 7.5 inch)
- 배경색: #F7F8FC
- 메인 텍스트: #111827
- 보조 텍스트: #6B7280
- 포인트 컬러: #4F7CFF (배지, 강조선, 칩)
- 카드 배경: 흰색(#FFFFFF), 모서리 둥글게, 그림자 처리
- 폰트: 한국어 기준 맑은 고딕

### 슬라이드 1: 사용자 현황 — 누가 이 시스템을 쓰는가
구성:
- 상단 타이틀: "크래프톤 그룹웨어를 사용하는 5가지 사용자 여정"
- 서브카피: "역할이 다르면 마찰도 다르다 — 페르소나별 핵심 경험 구조"
- 상단 키워드 칩 5개: 실무자 / 관리자 / 인사담당자 / 외주파트너 / 예비입사자
- 중앙 영역: 페르소나 카드 5개 (가로 배열)
  - 각 카드: 번호 배지(P1~P5) + 역할명 + 핵심 인용문(이탤릭) + 핵심 Pain 1줄
  - 카드별 포인트 컬러 구분:
    P1=#3B82F6 / P2=#10B981 / P3=#8B5CF6 / P4=#D97706 / P5=#DC2626
- 하단 인사이트 바: "동일한 시스템, 다른 여정 — 역할 맥락에 맞는 경험 설계가 필요합니다"

### 슬라이드 2: 핵심 마찰 구조와 설계 방향 — 어디서 끊기고 어떻게 연결할 것인가
구성:
- 상단 타이틀: "반복되는 시스템 전환이 만드는 마찰 — 그리고 통합 설계의 기회"
- 서브카피: "그룹웨어와 외부 SaaS 사이의 맥락 단절이 핵심 Pain이며, 통합이 핵심 해법입니다"
- 상단 키워드 칩 3개: 맥락 단절 / 통합 설계 / 자동화
- 중앙 카드 3개 (가로 배열):
  카드 1: "시스템 전환 비용"
    - 그룹웨어 ↔ 외부 Workflow SaaS 전환 반복
    - 결재 상태를 두 시스템에서 이중 확인
    - 인사 데이터 자동 연동 없는 수동 입력
    - 하단 인사이트: "맥락은 전환될 때마다 끊긴다"
  카드 2: "역할 맥락 부재"
    - 관리자와 실무자가 동일한 홈 화면 사용
    - Admin 역할 혼재로 탐색 비용 증가
    - 외주파트너·예비입사자 전용 진입 경로 없음
    - 하단 인사이트: "같은 화면, 다른 필요 — 역할 뷰가 없다"
  카드 3: "UX 설계 방향"
    - Workflow 그룹웨어 통합 → 시스템 이탈 제거
    - 역할 기반 홈 뷰 분리 (실무자/관리자/Admin)
    - 온보딩 자동화 → 예비입사자·외주파트너 공식 진입 경로
    - 하단 인사이트: "통합은 기능이 아니라 경험의 문제다"
- 하단 인사이트 바: "단일 시스템 안에서 역할별 맥락을 완결하는 것 — 그것이 이번 리뉴얼의 핵심 설계 목표입니다"

### 시각 요소 필수 포함
- 각 카드에 번호 배지 또는 아이콘 영역
- 카드 간 얇은 그라디언트 연결선 (화살표 또는 점선)
- 상단 키워드 칩 (둥근 배지 스타일)
- 하단 인사이트 바 (배경색 #111827, 텍스트 흰색)
- 도형 요소: 카드 배경에 은은한 원형 블러 오브젝트 (투명도 낮게)
