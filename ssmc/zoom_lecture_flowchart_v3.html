v3 변경사항 — 2026-03-05

1
start_url에 ZAK 포함 확인 — 별도 ZAK 발급 단계 제거
기존: ZAK Token API 별도 호출 → start_url 조합
변경: POST /v2/users/{email}/meetings 응답의 start_url에 ZAK가 이미 포함됨. 별도 발급 불필요
2
계정 가용성 판단 — type=upcoming 제거, type=live로 변경
기존: GET /v2/users/{userId}/meetings?type=upcoming → 미팅 없으면 배정 가능 판단
변경: Zoom은 미팅 생성 수 제한 없음 → upcoming으로 판단 불가. DB가 최종 판단, API type=live는 보조 참고용
01
📝강의 개설 요청
강사가 F-I-02 강의 개설 화면에서 강의명, 일시, 과정 입력 후 [개설하기] 클릭.
어드민 서버로 강의 개설 요청 전달.
02
🔍배정 계정 조회 (DB → API 보조)v3 수정
DB가 Source of Truth. API는 보조 참고용.

① DB 체크 (1차 판단)
zoom_accounts 테이블에서 assigned_course_id 확인 → 미배정 계정 선택
-- 가용 계정 조회 SELECT * FROM zoom_accounts WHERE assigned_course_id IS NULL AND status = 'active' LIMIT 1;
② Zoom API (보조 참고용)
// ✅ v3: type=live 사용 (live 중인 미팅 있으면 사용 중) GET /v2/users/{userId}/meetings?type=live // ❌ upcoming 사용 금지 // Zoom은 미팅 생성 수 제한 없음 → upcoming 유무로 배정 판단 불가
⚠️ DB 배정 상태와 API live 상태가 불일치할 경우 DB 우선. API는 참고용으로만 활용.
✅ 가용 계정 있음
Step 03으로 진행
❌ 가용 계정 없음
예비 계정 투입 또는 오류 반환. 강사에게 알림.
03
🔐Access Token 발급
Server-to-Server OAuth로 배정된 Zoom 계정의 Access Token 발급.
POST https://zoom.us/oauth/token ?grant_type=account_credentials &account_id={ACCOUNT_ID} // Authorization: Basic base64(CLIENT_ID:CLIENT_SECRET) // 응답: access_token (유효 1시간)
04
🎬미팅 생성 — start_url에 ZAK 포함v3 수정
POST /v2/users/{zoomAccountEmail}/meetings 호출.
POST /v2/users/{zoom_account_email}/meetings Authorization: Bearer {access_token} { "topic": "강의명", "type": 2, "start_time": "2026-03-10T10:00:00Z", "duration": 90, "settings": { "host_video": true, "participant_video": false, "auto_recording": "cloud" } }
응답 — start_url에 ZAK 자동 포함
{ "id": "95650366045", "join_url": "https://zoom.us/j/95650366045", "start_url": "https://zoom.us/s/95650366045?zak=eyJ0eXAi...", "password": "abc123" } // ✅ v3: start_url에 ZAK 포함 확인 // 별도 ZAK Token API 호출 불필요 // 웹/앱 모두 접속 가능한 링크
✅ start_url의 zak= 파라미터가 호스트 인증 역할. 강사 입장 시 이 URL 그대로 전달하면 됨.
05
💾DB 저장
미팅 생성 결과를 DB에 저장. start_url은 호스트 전용이므로 노출 주의.
-- lectures 테이블 저장 항목 meeting_id = "95650366045" join_url = "https://zoom.us/j/95650366045" password = "abc123" start_url = "https://zoom.us/s/...?zak=..." -- ✅ v3 추가 course_id = {course_id} -- 레코딩 귀속 추적 lecture_id = {lecture_id} -- 레코딩 귀속 추적 lecture_status = "scheduled"
⚠️ start_url은 호스트 전용 URL. 수강생에게 노출 금지. API 응답으로 반환 시 마스킹 처리.
💡 방안 A: 강의 시작 버튼 클릭 시 GET /v2/meetings/{meetingId} 재조회 → start_url 재발급 (ZAK 만료 대비)
06
👨‍🏫강사 입장 (F-I-04)
강사가 [강의 시작] 클릭 → start_url (ZAK 포함) 직접 오픈.
// ✅ v3: start_url에 ZAK 포함 → 별도 ZAK 발급 불필요 window.open(start_url, '_blank'); // start_url = "https://zoom.us/s/{meetingId}?zak=..." // 웹 브라우저: Zoom Web Client 실행 // Zoom 앱 설치 시: 앱으로 자동 실행
앱 미설치 시 흐름
앱 설치됨
zoomus:// 딥링크 or start_url 클릭 → 앱 자동 실행
앱 미설치
안내 모달 → 다운로드 링크 → [다시 시도] 버튼 or 웹으로 계속
강의 시작 시 lecture_status = "active" 업데이트 → 수강생 버튼 활성화
07
👥수강생 입장 (F-S-02)
강사 시작 전: 버튼 비활성화 (lecture_status ≠ active)
강사 시작 후: join_url 새 탭으로 열기 — Zoom Web Client 기본
// 수강생은 Signature 생성 불필요 window.open(join_url, '_blank'); // join_url = "https://zoom.us/j/{meetingId}"
08
