1. 프로젝트 개요

목적AI Studio LMS에 Zoom 기반 실시간 강의 기능 통합. 강사 화면 공유 및 실시간 상호작용 환경 제공.
인증 방식Server-to-Server OAuth — 마스터 계정 단일 앱 확정
마스터 계정aistudio.service@multicampus.com
자격증명서버 .env — ZOOM_ACCOUNT_ID / ZOOM_CLIENT_ID / ZOOM_CLIENT_SECRET
API Scopemeeting:write:admin, user:read:admin
사용자 역할강사 / 운영자 / 수강생 / 어드민

3. 강의 모드 (2026-03-17 확정)

실시간 모드

단계내용
트리거강사 [지금 시작] 클릭
처리가용 계정 즉시 배정 → start_url 즉시 발급
버퍼없음 — 즉시 시작, 즉시 반납
예외가용 계정 없을 시: "현재 사용 가능한 계정이 없습니다" 인라인 에러

예약 모드

단계내용
트리거강사 [예약하기] 클릭
범위 (1차)당일 예약만 — 날짜 선택 불가 (고정)
입력강의명 / 시작 시간 select / 종료 시간 select
시간 단위30분 단위 (개발팀 협의)
최소 강의 시간1시간 — 종료 시간은 시작 + 1시간 이상만 활성화
버퍼 정책앞뒤 각 30분 서버 자동 확보 (UI 노출 안 함)
슬롯 가용성서버 API가 가능/불가 시간 계산 → 프론트는 disabled 처리만
원활한 강의 진행을 위해 시작 30분 전·종료 30분 후 시간이 자동으로 예약됩니다. (강사 안내 문구 확정)

예외처리 케이스

케이스처리
버퍼 충돌"해당 시간 예약 가능한 계정이 없습니다" 인라인 에러
가용 계정 없음"현재 예약 가능한 계정이 없습니다 + 관리자 문의 안내"
동시 신청 충돌DB 낙관적 락 → 늦은 쪽 "방금 다른 강의가 배정됐습니다. 시간을 다시 선택해주세요"
자동연장 충돌운영자에게 push 알림 발송
예약 취소버퍼 포함 계정 즉시 반납 → 슬롯 재렌더링
강제종료 정책TBD — 현업 운영자 미팅 후 확정

4. 녹화 & 다시보기

항목결정상태
저장 방식Zoom 클라우드 저장 유지 (강제 녹화 설정)확정
S3 이관개발팀 확인 중 (만료 케이스 한정 검토)확인 중
다시보기 방식Zoom 클라우드 링크(share_url)로 이동 — 임베딩 안 함확정
Webhook 흐름recording.completed → Secret Token 검증 → share_url DB 저장 → 강사 지난강의 목록에 노출확정
녹화 미완료 케이스용량 초과 등 예외 케이스 — 1차 범위 제외확정
클라우드 용량Pro/Business 기준 계정당 10GB — 관리 정책 별도 수립 필요정책 필요

buffer_end버퍼 포함 점유 종료 (reserved_end + 30분)

6. 성능 & 제약사항

항목내용
동시 접속자라이선스 유형별 100~300명. 대규모 세미나는 Webinar 라이선스 별도 필요.
